# Contributing to Pickleball Platform

Thank you for your interest in contributing! Here's how to get started.

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `npm install` in both `frontend/` and `backend/`
3. Copy `.env.example` to `.env` and configure your local PostgreSQL
4. Run `npm run dev` to start both servers

## Branch Naming

- `feature/<description>` — New features
- `fix/<description>` — Bug fixes
- `docs/<description>` — Documentation updates

## Pull Request Process

1. Create a feature branch from `master`
2. Make your changes with clear, descriptive commits
3. Ensure the app runs locally without errors
4. Update documentation if you changed any APIs or behavior
5. Submit a PR with a clear description of what and why

## Code Style

- Use consistent formatting (Prettier defaults)
- Meaningful variable and function names
- Comment complex logic, not obvious code

## Reporting Issues

Open a GitHub Issue with:
- Clear title and description
- Steps to reproduce (for bugs)
- Expected vs actual behavior
- Screenshots if applicable
